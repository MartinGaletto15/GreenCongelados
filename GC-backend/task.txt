ADDRESS

CATEGORY
Post category --> Verificar que sea administrador/superadministrador

Get category --> Disponible para todos
Put category --> Verificar que sea administrador/superadministrador
Delete category --> Verificar que sea administrador/superadministrador

ORDER
Post order:
    a - Guardar el ID del usuario que genera el pedido, la fecha actual y Verificar el codigo de promocion
    b - Verificar el shipping cost y guardarlo
    c - Guardar campos de envio asociados al usuario
    d - Guardar promociones y total
    e - Inicializar estado en pendiente
Get order by id --> Verificar que el usuario sea dueño de esa orden o que sea Administrador/Cadete
Get order --> Verificar que sea Administrador/superadministrador
Get orderMe --> Agregar
Put order --> Verificar que esa orden sea del usuario y que no este en estado entregado
put orderStatus --> Verificar que sea Administrador/Cadete
Delete order --> Debe funcionar como un cambio de estado a cancelado
